Building a Secure Web Application for Uploading Files to Azure Blob Storage with Time-Limited Shareable Links

In today’s cloud-focused environment, it’s really important to manage file uploads safely and share them with the right people. Whether you’re working on a big project for work or just a simple personal website, knowing how to securely upload files and create links that expire after a while is a useful skill. In this guide, we’ll go through how to build a web application that lets users upload files to Azure Blob Storage securely and creates special links that only certain users can use for a limited time. As I continue my cloud journey I decided to create a web application for securely uploading files to Azure Blob Storage with time-limited, shareable links. The aim of this project is to build a web application where:

Key Features

  • Users can upload files securely to Azure Blob Storage.
  • The application generates unique, time-limited links for users to share, ensuring that only authorized users with the link can access the file during the specified duration.
  • The application is built using ASP.NET Core, Azure Blob Storage, and Azure SAS (Shared Access Signature) tokens for secure file access control.

Prerequisites

In this project, the following services were used:

  • Azure AD
  • Azure Logic Apps
  • Azure Email Service
  • Azure Resource Manager

Azure Active Directory

Azure Active Directory acts as the central directory for your organization's users and their access permissions. You configure it with user attributes, roles, and groups relevant to your organization. I already had one set up, so not much configuring was necessary here.

Azure Logic App Workflow

Azure Logic Apps is a cloud service offered by Microsoft Azure that allows you to create and run workflows for automating business processes and integrating various services and applications. Using Logic Apps and Entra ID, I have built a workflow that will automatically trigger whenever a new entry is added to Sharepoint, or a specific email or form is submitted. This way I could test and show that the automation is working. When the trigger event occurs, the Logic App workflow is initiated. The first action was to create a user in Microsoft Entra ID from the HTTP request , added a new parameter for Job Title which I used to assigned the user to the right group
